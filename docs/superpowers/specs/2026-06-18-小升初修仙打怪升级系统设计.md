# 小升初修仙打怪升级系统 — 设计文档

> 版本：v1.0 | 日期：2026-06-18 | 状态：待审阅

---

## 一、背景与目标

### 1.1 背景

小学毕业生即将进入苏州市振华初级中学，需要利用暑假（6 月 25 日 — 8 月 25 日，约 62 天）自学完成初一语、数、外三门课程。学习过程需要被量化、激励和追踪，避免松散和半途而废。

### 1.2 目标

打造一款**网页版「修仙打怪」学习系统**，把枯燥的自学变成击败妖兽、积累修为、突破境界的闯关体验，帮助孩子在两个月内完成初一衔接。

### 1.3 成功标准

- 学生每天打开系统，能在 5 秒内知道今天要击败哪些「妖兽」（学习任务）。
- 语数外三科在 62 天内完成一轮系统学习。
- 英语中考高频词按主题词群滚动学习，并通过艾宾浩斯复习巩固。
- 支持自定义任务（如体育锻炼）和临时任务，学习与生活目标统一。
- 数据可本地备份，换设备或清浏览器后能通过 JSON 文件恢复。

## 二、产品形态

### 2.1 形态

- **Web 单页应用（SPA）**，桌面端优先，响应式适配移动端。
- 无需独立服务器，无需购买域名。
- 构建后部署到 GitHub Pages / Vercel / Netlify 等免费静态托管平台。

### 2.2 技术栈

| 层级 | 选型 |
| --- | --- |
| 框架 | React 19 + TypeScript |
| 构建工具 | Vite |
| 样式 | TailwindCSS 3.4 |
| 路由 | React Router DOM v6 |
| 图标 | Remix Icon / FontAwesome（CDN） |
| 状态 | React useState + useEffect |
| 持久化 | localStorage + JSON 文件导入/导出 |
| 字体 | Google Fonts（中文优先可读性） |

## 三、核心概念（修仙世界观）

| 真实含义 | 修仙包装 |
| --- | --- |
| 学科（语/数/外） | 三大功法 |
| 知识点 / 单词 | 妖兽 / 凶兽 |
| 完成学习任务 | 击败妖兽 |
| 连续打卡 | 闭关天数 |
| 学习时长 | 修为积累 |
| 完成阶段性目标 | 突破境界 |
| 周末综合练习 | 宗门试炼 |
| 62 天结束 | 渡劫飞升 |
| 导出/导入数据 | 导出/导入修为档案 |

### 3.1 境界体系

按累计修为划分境界，作为长期目标：

| 境界 | 累计修为 |
| --- | --- |
| 凡人 | 0 |
| 炼气 | 60 |
| 筑基 | 300 |
| 金丹 | 900 |
| 元婴 | 1800 |
| 化神 | 3600 |
| 渡劫 | 7200 |

修为计算：学习 1 分钟 = 1 点修为，完成任务额外 +10 点，连续打卡有加成。

## 四、功能模块

### 4.1 洞府首页（Home）

**目标**：一眼看清今天要做什么、修为到哪了、还需复习什么。

| 模块 | 内容 |
| --- | --- |
| 顶部横幅 | 问候语 + 今日修为目标 + 「开始历练」按钮 |
| 状态面板 | 当前境界、今日修为、连续闭关天数、今日击败数 |
| 今日妖兽列表 | 按功法分组，显示任务名、类型、预计时长、完成进度 |
| 功法进度 | 三科各一张卡片，显示击败进度百分比 |
| 复习提醒 | 艾宾浩斯复习列表，显示今日需复习内容 |
| 本周战绩 | 柱状图，每日已完成 vs 未完成 |

### 4.2 今日历练（Daily）

**目标**：当天任务的唯一执行入口，专注完成、即时反馈。

| 模块 | 内容 |
| --- | --- |
| 日期标题 | 当前日期 + 随机修仙激励语 |
| 任务列表 | 按功法分组，每组含任务血条、击败按钮 |
| 专注计时 | 圆形计时器，预设 15/25/45 分钟 |
| 击败动画 | 完成任务时播放妖兽倒下 + 修为增加动画 |
| 自动跳转 | 击败当前妖兽后自动展开下一个未完成任务 |
| 全部完成 | 显示「今日历练圆满」庆祝卡片 |

### 4.3 功法总纲（Plan）

**目标**：62 天整体排期，可切换学科查看，可调整计划模式。

| 模块 | 内容 |
| --- | --- |
| 学科统计 | 三科已完成/总数 + 进度条 |
| 计划模式 | 标准模式（每天 3 科，每科 40 分钟）、周末试炼模式 |
| 日历排期 | Day 1 ~ Day 62，每天显示任务清单 |
| 学科筛选 | 全部 / 英语 / 语文 / 数学 |
| 手动调整 | 拖拽/移动任务到天，系统重新计算后续排期 |

**62 天排期原则**：

- 英语：每天 10-12 个新词（主题词群）+ 艾宾浩斯复习；周末加入综合复习。
- 语文：每天 1 篇课文/古诗/阅读片段 或 1 个写作素材主题。
- 数学：每天 1-2 个知识点 + 2-3 道配套练习；周末加入章节综合练习。

### 4.4 三科功法（Subject）

#### 英语功法

| 章节 | 课时数 | 内容 |
| --- | --- | --- |
| 主题词群 | 约 40 | 按场景/话题分组的中考高频词，如校园生活、家庭、食物、旅行 |
| 音标修炼 | 5 | 单元音、双元音、清辅音、浊辅音、综合练习 |
| 语法关卡 | 8 | 初一基础语法点 |
| 艾宾浩斯复习 | 按遗忘曲线 | 8 轮自动复习 |

**词汇学习方式**：

- 不按字母顺序，按主题词群串联。
- 每个词含：单词、音标、词性、释义、例句、相关词（同义/反义/同话题）。
- 默认词表约 800 词，支持用户导入自定义词表。

#### 语文功法

| 章节 | 课时数 | 内容 |
| --- | --- | --- |
| 课文精讲 | 3 | 《春》《济南的冬天》《雨的四季》 |
| 古诗文背诵 | 6 | 初一必背古诗词 |
| 阅读理解 | 4 | 经典散文/记叙文阅读 + 选择题 |
| 写作素材 | 12 | 人物、景物、修辞、开头结尾、名言积累 |

#### 数学功法

| 章节 | 知识点数 | 内容 |
| --- | --- | --- |
| 有理数 | 12 | 正负数、数轴、相反数、绝对值、四则运算、乘方、科学记数法 |
| 整式 | 10 | 代数式、单项式、多项式、合并同类项、去括号 |
| 一元一次方程 | 10 | 方程概念、等式性质、移项、去括号、去分母、应用题 |

每个知识点含：概念讲解、典型例题、1-3 道即时练习。

### 4.5 试炼记录（Records）

**目标**：追踪学习历史、展现成长曲线。

| 模块 | 内容 |
| --- | --- |
| 统计面板 | 累计闭关天数、累计修为、任务完成率、当前连续闭关 |
| 打卡日历 | 月视图，绿色=圆满、浅绿=部分、灰色=未闭关 |
| 学习趋势 | 每周击败妖兽数 / 每周修为 |
| 功法时间分布 | 三科投入占比 |
| 成就徽章 | 首次闭关、7 天、30 天、击败 100 妖兽、累计 1000 修为、全科完成 |
| 境界突破 | 显示当前境界和下一阶段 |

### 4.6 设置（Settings）

| 模块 | 内容 |
| --- | --- |
| 自定义任务 | 名称、时长、优先级、重复规则 |
| 临时任务 | 快速添加到今日，可选加入固定任务 |
| 词表导入 | 上传自定义英语词表（CSV/JSON） |
| 数据备份 | 导出修为档案（JSON）、导入修为档案 |
| 计划重置 | 重置所有进度（二次确认） |

## 五、数据模型

### 5.1 存储策略

- **localStorage**：学习进度、任务状态、用户偏好、自定义任务。
- **JSON 文件导入/导出**：跨设备备份与恢复。
- **Mock 数据文件（`src/mocks/`）**：课程内容、任务模板、统计数据。

### 5.2 localStorage 键值

| Key | 类型 | 说明 |
| --- | --- | --- |
| `cultivation-progress` | object | 各科完成进度 |
| `daily-records` | object[] | 每日打卡记录 |
| `custom-tasks` | object[] | 自定义任务 |
| `user-preferences` | object | 计划模式、通知等偏好 |
| `english-vocab-custom` | object[] | 用户导入的自定义词表 |

### 5.3 核心数据结构

#### 任务（Task）

```typescript
interface Task {
  id: string;
  title: string;           // 妖兽名
  subject: 'english' | 'chinese' | 'math' | 'custom';
  type: 'learn' | 'review' | 'practice';
  duration: number;        // 预计分钟
  priority: 'low' | 'medium' | 'high';
  repeatRule?: 'none' | 'daily' | 'weekly';
  completed: boolean;
  completedAt?: string;    // ISO date
  source: 'auto' | 'manual' | 'temp';
  contentId?: string;      // 关联课程内容
}
```

#### 每日记录（DailyRecord）

```typescript
interface DailyRecord {
  date: string;            // YYYY-MM-DD
  tasks: Task[];
  totalDuration: number;   // 实际学习分钟
  cultivation: number;     // 获得修为
  completedCount: number;
  streakContinued: boolean;
}
```

#### 学习进度（Progress）

```typescript
interface Progress {
  english: string[];       // 已完成 contentId 数组
  chinese: string[];
  math: string[];
  totalCultivation: number;
  currentRealm: string;
}
```

## 六、游戏化设计

### 6.1 妖兽血条

每个学习任务对应一只妖兽，显示血条：

- 未开始：100% 血量
- 进行中：根据投入时长下降
- 击败：0% 血量 + 击败动画

### 6.2 连击系统

- 连续击败任务获得连击加成，连击 ≥3 时修为额外 +10%。
- 连击 ≥5 时触发「顿悟」提示。

### 6.3 境界突破

累计修为达到境界阈值时：

- 显示「突破境界」动画。
- 解锁对应称号（如「筑基期修士」）。

### 6.4 宗门试炼（周末综合）

每周六、日触发综合练习：

- 英语：本周主题词群听写/选择。
- 语文：本周古诗默写/阅读理解。
- 数学：本周知识点综合题。

## 七、复习机制

采用**艾宾浩斯遗忘曲线**，自动安排复习：

- 学过的内容会在第 1、2、4、7、15 天自动出现在复习列表。
- 复习任务优先于新学任务显示。
- 复习完成也击败妖兽，获得修为。

---

## 八、自定义任务与临时任务

### 8.1 自定义任务

- 属性：名称、时长、优先级、重复规则。
- 重复任务每天自动生成到当日列表。
- 可暂停/删除。

### 8.2 临时任务

- 默认只出现在今天。
- 完成后可选择「加入固定任务」。
- 未完成则次日自动消失。

## 九、部署与数据安全

### 9.1 部署方式

1. 本地开发：`npm run dev`
2. 构建：`npm run build`
3. 部署：将 `dist/` 目录上传到 GitHub Pages / Vercel / Netlify。

### 9.2 数据安全

- 所有数据只存在用户浏览器中，不上传到任何服务器。
- 建议每周「导出修为档案」备份一次。
- 清除浏览器数据前务必先导出备份。

## 十、内容策略

### 10.1 教材内容

- 以苏州市振华初级中学教材方向为纲，但具体课文/知识点采用通用初一内容。
- 语文课文、数学知识点、英语主题词群均可由用户在 `src/mocks/` 中替换或扩展。
- 提供标准模板，方便后续导入真实教材内容。

### 10.2 英语词表

- 内置约 800 个中考高频词，按 40 个主题词群组织。
- 每个词群 15-20 词，每天学习 10-12 个新词。
- 支持用户通过 CSV/JSON 导入自定义词表。

## 十一、风险与假设

| 风险 | 应对 |
| --- | --- |
| 两个月内容量过大 | 系统按 62 天自动均摊，用户可调整每日强度 |
| 学生坚持不下来 | 游戏化反馈 + 连续闭关 + 成就徽章 |
| 数据丢失 | 每周导出备份提醒 |
| 教材内容不匹配 | 预留自定义内容入口，用户可替换 |
| 移动端体验差 | 响应式设计，优先保证核心流程可用 |
