# 初一衔接学习系统 — 产品需求文档（PRD）

> 版本：v2.0 | 更新时间：2026-06-18 | 状态：已完成交付

---

## 一、产品概述

### 1.1 产品定位

「初一衔接助手」是一款面向小升初学生的在线衔接学习工具，覆盖英语、语文、数学三门核心学科，提供体系化的课程内容、每日学习打卡、学习进度追踪与成长数据分析。产品核心价值：帮助学生在暑假实现小学到初中的无缝衔接，养成良好学习习惯。

### 1.2 目标用户

- **主要用户**：即将升入初一的小学六年级毕业生（12-13岁）
    
- **次要用户**：关心孩子学习进度的家长
    
- **使用场景**：暑期每日在家自学，配合家长监督
    

### 1.3 产品形态

Web 单页应用（SPA），桌面端优先，响应式适配移动端。

## 二、页面结构与导航

### 2.1 页面路由

|路由|页面|功能定位|
|---|---|---|
|`/`|学习首页|仪表盘：今日概览、学科进度、复习提醒、本周统计|
|`/plan`|学习计划|任务自动拆解、学科筛选、计划模式切换|
|`/daily`|每日清单|今日任务执行、专注计时器、标记完成自动跳转|
|`/subject/english`|英语课程|大纲侧边栏 + 逐项学习（21个课时）|
|`/subject/chinese`|语文课程|大纲侧边栏 + 逐项学习（25个课时）|
|`/subject/math`|数学课程|大纲侧边栏 + 逐项学习（32个知识点）|
|`/records`|学习记录|打卡日历、时长趋势、学科分布、成就徽章、学习分析|

### 2.2 导航设计

- **桌面端**：顶部固定导航栏，品牌 Logo + 7个导航链接，当前页高亮
    
- **移动端**：底部固定 Tab 栏（前5个）+ 右上角汉堡菜单
    
- **学科页面内**：左侧可折叠大纲侧边栏（256px宽），右侧内容区 + 浮动折叠按钮

## 三、功能详述

### 3.1 学习首页（Dashboard）

**功能目标**：登录后一眼看清今天要做什么、各科学到哪了、还需复习什么。

**功能模块：**

| 模块       | 内容                                                     |
| -------- | ------------------------------------------------------ |
| 顶部横幅     | 早安问候语 + 随机学习格言 + 「开始学习」按钮（直达每日清单并自动展开第一个待办）            |
| 统计卡片（4格） | 连续打卡天数（含最长记录）、今日完成数/总数（含进度条）、今日已学/剩余时长、整体学习进度          |
| 今日任务列表   | 每项显示：学科标签（颜色区分）、任务名、类型、时长、完成进度数。点击圆圈切换完成状态             |
| 学科进度卡片   | 三科各一张，显示完成课时数/总数 + 百分比进度条。点击展开显示每个章节的细粒度进度条 + 「进入课程」链接 |
| 本周统计     | 柱状图，每日已完成（主题色）+ 未完成（灰色），底部图例                           |
| 复习提醒     | 基于艾宾浩斯记忆曲线的复习提醒卡片，显示内容、时间、学科标签                         |

**交互规则：**

- 任务点击圆圈 → 切换完成/未完成，数据实时更新
    
- 学科进度卡片点击 → 展开/收起章节明细
    
- 「开始学习」按钮 → 跳转 `/daily?start=first`，自动展开首个未完成任务并滚动至其位置
    

---

### 3.2 学习计划（Plan）

**功能目标**：按「天数 × 学科」自动拆解学习任务，用户可切换学科查看。

**功能模块：**

|模块|内容|
|---|---|
|学科统计卡片|三科各一张，显示已完成/总任务数 + 进度条 + 百分比|
|计划模式切换|三种模式：标准模式（每天3科各15分钟）、冲刺模式（每天3科各30分钟）、周末模式（仅英语+数学）|
|任务清单|按天分组（Day 1 ~ Day 7），每天显示学科任务列表。支持全部/英语/语文/数学筛选|
|总体进度条|顶部进度条显示整体完成百分比|

**交互规则：**

- Tab 切换器（全部/英语/语文/数学）→ 筛选显示对应学科任务
    
- 计划模式按钮 → 切换模板
    
- 「去打卡」按钮 → 跳转每日清单
    

---

### 3.3 每日学习清单（Daily）

**功能目标**：当日任务的唯一执行入口，支持展开查看学习内容、专注计时、一键完成跳转下一个。

**功能模块：**

|模块|内容|
|---|---|
|日期标题|当前日期 + 随机激励语|
|统计卡片|任务完成数/总数、已学时长/总时长、专注计时器状态|
|任务列表（主区域）|按学科分组，每组：学科标题栏（含全部完成按钮） + 任务列表|
|专注计时器（侧边栏）|圆形计时器 + 3个预设时长（15分/25分/45分） + 结束按钮|
|进度汇总|完成进度条 + 时间投入进度条 + 智能学习建议|

**展开任务后的内容渲染规则（按类型）：**

|任务类型|展示形式|
|---|---|
|词汇学习|3-4列翻转卡片网格，正面单词+音标，背面释义+例句+词性|
|音标学习|3列卡片，每个音标显示符号+示例词+发音要点|
|课文分析|原文段落 + 批注卡片（灯泡图标+注解）|
|古诗学习|诗词正文 + 名句标签 + 可展开的赏析区域|
|写作素材|3列卡片，分类图标+标题+内容+来源|
|知识点学习|编号列表，点击展开详细讲解+要点列表|
|练习题|题目卡片 + 难度标签 + 「查看答案」按钮 → 答案揭晓|
|艾宾浩斯复习|翻转卡片（正面提示 → 反面答案）或练习题|

**每个展开任务底部的操作栏：**

- 「标记完成，继续下一个」按钮（仅未完成任务显示）→ 当前任务标记完成并收起 → 自动展开下一个未完成任务 → 页面滚动至其位置
    
- 「去XX课程页面深入学习 →」链接 → 跳转对应学科课程页面
    

**全部完成后：**

- 显示庆祝卡片「太棒了！今日任务全部完成！」

### 3.4 英语课程（Subject/English）

**功能目标**：21个课时的完整英语学习课程，左侧大纲导航 + 右侧逐项学习。

**课程大纲结构（4章，21课时）：**

|章节|图标|课时数|内容|
|---|---|---|---|
|词汇积累|📖|6|Unit 1-6，每单元10词，共60词|
|音标学习|🗣️|5|单元音、双元音、清辅音、浊辅音与鼻音、综合练习|
|拼写训练|✏️|1|15个单词 + 连击系统 + 字母槽位揭晓|
|艾宾浩斯复习|🔄|8|按遗忘曲线8轮复习计划|

**内容展示：**

|类型|展示形式|
|---|---|
|词汇|4列翻转卡片，已掌握单词有绿色标记，点击翻转看释义|
|音标|3列卡片，每个音标显示符号、示例词、发音要点|
|拼写|2列输入卡片，输入框 + 检查按钮、「当前连击/最佳连击」统计、连击≥5弹出庆祝动画、字母槽位揭晓|
|复习|艾宾浩斯原理提示卡片 + 任务清单（已完成项打勾）|

**交互规则：**

- 左侧大纲点击任一项 → 右侧切换对应内容
    
- 底部「标记为已学完」按钮 → 该项在大纲中显示打勾，进度条更新
    
- 「上一项/下一项」按钮 → 按课程顺序翻页
    
- 侧边栏折叠按钮 → 大纲收起/展开
    
- 所有完成状态持久化到 `localStorage`（key: `eng-progress`）

### 3.6 数学课程（Subject/Math）

**功能目标**：32个知识点的完整数学学习课程 + 18道配套练习题 + 每日目标。

**课程大纲结构（3章，32个知识点）：**

|章节|知识点数|内容范围|
|---|---|---|
|有理数|12|正负数、数轴、相反数、绝对值、有理数加减乘除、乘方、科学记数法等|
|整式|10|代数式、单项式、多项式、整式加减、合并同类项、去括号法则等|
|一元一次方程|10|方程概念、等式性质、移项、去括号解方程、去分母、应用题等|

**每个知识点的展示结构：**

|区域|内容|
|---|---|
|顶部标签|章节名称 + 状态标签（已掌握/学习中/未开始）|
|知识点标题+描述|一句话简介|
|详细讲解卡片|书籍图标 + 完整概念讲解|
|配套例题卡片|灯泡图标 + 典型例题及解答（黄色底）|

**练习题系统：**

- 18道练习题，按当前所在章节自动筛选（有理数6道/整式6道/一元一次方程6道）
    
- 每道题标注难度（简单/中等/较难）
    
- 「查看答案」按钮 → 答案揭晓区域（可再次隐藏）
    

**每日目标模块：**

- 页面底部展示当前章节的每日目标
    
- 每条目标显示日期、目标描述、进度条（已完成题数/目标题数）、完成状态图标
    

**交互规则：**

- 大纲中每个知识点右侧显示状态标签（已掌握/学习中/未开始）
    
- 「查看本章练习题」按钮 → 展开/收起练习题列表
    
- 「标记为已掌握」按钮 → 状态更新
    
- 其余同上（侧边栏折叠、上一项/下一项）
    
- 完成状态持久化到 `localStorage`（key: `math-progress`）

**功能模块：**

## 四、数据模型

## 六、关键交互规范

### 6.1 学习闭环

```
首页概览 → 学习计划（规划） → 每日清单（执行） → 学科课程（深入学习） → 学习记录（复盘）
```

### 6.2 进度持久化规则

- 所有课程进度（标记完成/已掌握）立即写入 `localStorage`
    
- 页面刷新后自动从 `localStorage` 恢复
    
- 初始状态（首次访问）从 Mock 数据中读取默认已完成项
    

### 6.3 标记完成跳转逻辑（每日清单）

1. 用户点击「标记完成，继续下一个」
    
2. 当前任务标记为已完成（done 更新为 total）
    
3. 当前任务展开区域收起
    
4. 查找下一个未完成任务的 ID
    
5. 若找到 → 展开该任务 → 350ms 后页面平滑滚动至其位置
    
6. 若未找到（已是最后一个）→ 显示全部完成庆祝卡片
    

### 6.4 响应式适配规则

- 桌面端优先设计（≥1024px）
    
- 统计卡片：`grid-cols-1 sm:grid-cols-2 lg:grid-cols-4`
    
- 内容区：`grid-cols-1 lg:grid-cols-3`（2:1 比例）
    
- 导航栏：桌面 `md:flex` 横向，移动端底部固定 Tab
    
- 侧边栏（课程页）：256px 固定宽，移动端折叠
